## Structure

The Autocomplete component consists of three elements:

- **post-autocomplete**: The main container that manages the combobox behavior
- **post-listbox**: The dropdown panel containing the options
- **post-option**: Individual selectable options within the listbox

```html
<post-autocomplete>
  <label for="my-input" class="form-label">Select a country</label>
  <input id="my-input" class="form-control" type="text" placeholder="Type to search..." />
  <post-listbox>
    <post-option value="ch">Switzerland</post-option>
    <post-option value="de">Germany</post-option>
    <post-option value="fr">France</post-option>
  </post-listbox>
</post-autocomplete>
```

## Custom Filtering

By default, the autocomplete filters options by matching the input value against option labels (case-insensitive).
You can implement custom filtering by listening to the `postFilterRequest` event and calling `preventDefault()`.

## Keyboard Navigation

The autocomplete supports full keyboard navigation:

| Key | Action |
|-----|--------|
| **Arrow Down** | Open listbox (if closed) or move to next option |
| **Arrow Up** | Open listbox (if closed) or move to previous option |
| **Home** | Move to first option |
| **End** | Move to last option |
| **Enter** | Select the active option |
| **Escape** | Close the listbox |
| **Tab** | Close the listbox and move focus |

## Accessibility

The autocomplete implements the [ARIA Combobox pattern](https://www.w3.org/WAI/ARIA/apg/patterns/combobox/) with the following features:

- `role="combobox"` on the input element
- `aria-expanded` reflects the listbox open state
- `aria-activedescendant` points to the currently active option
- `aria-controls` links the input to the listbox
- Options have `role="option"` with appropriate `aria-selected` and `aria-disabled` states
