## SCSS Imports

To import all styles from the design-system-styles package:

```css dark
@use "@swisspost/design-system-styles/index.scss";
```

To import only the styles required for this component:

```css dark
@use "@swisspost/design-system-styles/basics.scss";
@use '@swisspost/design-system-styles/components/forms.scss';

// required if you use floating-labels
@use '@swisspost/design-system-styles/components/floating-label.scss';

// required if you use validation feedbacks
@use '@swisspost/design-system-styles/components/form-feedback.scss';
```

## Specific examples of application

The following examples show the different characteristics of the component. These can differ in the type of visualization, the html structure, as well as when, how and why they are displayed.

### Floating Label

Wrap a pair of `textarea` and `label` elements in a `.form-floating` container to enable floating labels.<br/>
But note that the `textarea` must come first, so we can ensure the correct styles.

### Sizing

The size can be changed by simply adding one of four classes:

- Small (`form-control-sm`)
- Regular (`form-control-rg`)
- Medium (default or `form-control-md`)
- Large (`form-control-lg`)

<div class="alert alert-info">

### Validation

Simply add the classes `is-valid` or `is-invalid` to the `textarea` element to show it in the expected state.<br/>
When the component has been validated, don't forget to add a `.valid-feedback` or `.invalid-feedback` element after the `textarea` for accessability reasons.
