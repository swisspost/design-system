The `<post-login-widget>` integrates with the Swiss Post authentication system (KLP). It automatically fetches the current session on load and renders the appropriate UI — a login link for unauthenticated users, or an avatar-triggered dropdown menu with account links for authenticated users.

## Usage in the Header

Place the widget in the `post-login` slot of `<post-header>`:

```html
<post-header text-menu="Menu">
  <post-login-widget
    slot="post-login"
    login-url="https://account.post.ch/login"
    logout-url="https://account.post.ch/logout"
    text-user-profile="My Profile"
    text-messages="Messages"
    text-settings="Settings"
    text-logout="Logout"
  ></post-login-widget>
</post-header>
```

## Logged Out

When no active session is detected, the widget renders a login link pointing to the `login-url` prop. On mobile the label is hidden and only the icon is shown.

## Logged In

When an active session is found, the widget renders a `<post-avatar>` inside a menu trigger. Clicking the avatar opens a dropdown containing the user's name, links to their profile, messages, and settings, and a logout button.

## Logout Handling

The widget does **not** perform the logout redirect itself. Instead, it emits a `postLogout` event with the `logout-url` as the payload, letting the consuming application handle the redirect:

```js
document.querySelector('post-login-widget').addEventListener('postLogout', (e) => {
  window.location.href = e.detail;
});
```

## Internationalization

All visible labels are configurable via props to support the four Swiss Post languages (German, French, Italian, English):

| Prop                | Description                     |
| ------------------- | ------------------------------- |
| `text-user-profile` | Label for the "My Profile" link |
| `text-messages`     | Label for the "Messages" link   |
| `text-settings`     | Label for the "Settings" link   |
| `text-logout`       | Label for the "Logout" button   |
