## Accessible alternative for essential content

When the additional information is essential and must be available to every user, including on
touch devices, use an information icon button that toggles a `<post-popover>` instead of a tooltip.
Unlike a tooltip, the popover is opened by an explicit click or key press, so it works consistently
across mouse, keyboard and touch input.

Keep in mind that this approach only works for elements that do not trigger an action of their own.
The information icon button is a separate interactive element, so it cannot be nested inside another
interactive element such as a button or link. If you need to add essential information to an element
that already triggers an action, place the information icon button next to it instead of inside it.

## Examples

### Tooltip on non-focusable elements

If you assign a tooltip to a non-focusable element, the component will add `tabindex="0"` automatically. With that, the tooltip is also visible for keyboard users. Also, if you forget to link the tooltip and the trigger with `aria-describedby="tooltip-id"`, we've got you covered.
