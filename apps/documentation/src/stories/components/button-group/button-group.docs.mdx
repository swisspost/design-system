## Usage examples

The following examples demonstrate the different variations of the component. These may differ in visual appearance, HTML structure, and usage context.

### Direction variants

A button group can be displayed either horizontally or vertically. Modifier classes are available for each breakpoint, allowing you to control when the layout switches between these orientations.

**Classes:**

- Horizontal (default): `.btn-group-{breakpoint}-horizontal`
- Vertical: `.btn-group-vertical` for all breakpoints, or `.btn-group-{breakpoint}-vertical`

The example below shows a vertical button group that switches to a horizontal layout at the large breakpoint.

<Canvas sourceState="hidden" of={ButtonGroupStories.Direction} />

### Checkboxes and radio buttons

A button group can also be used with checkable inputs. In these cases, the buttons behave as form controls and may support a disabled state.

All buttons are linked, allowing only one selection at a time.
