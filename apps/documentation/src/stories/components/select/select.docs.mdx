Our component customizes the native `<select>` with CSS that overrides its initial appearance.

Custom `<select>` menus need only the class `.form-select` to apply the custom styles. Due to browser limitations, these styles only affect the `<select>` element itself and cannot be extended to the `<option>` elements.

## Concrete Examples of Application

The following examples show the different characteristics of the component. These can differ in the type of visualization, the HTML structure, as well as when, how and why they are displayed.

### Floating Label

Wrap a pair of `<select>` and `<label>` elements in a `.form-floating` container to enable floating labels.<br/>
But note that the `<select>` must come first, so we can ensure the correct styles.
In order for the label to work as a placeholder, the select needs to have an empty option.

### Small select

Use `.form-select-sm` for a compact select. Floating labels are not supported with this variant.
