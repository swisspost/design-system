## Page structure

In order for your page content to not be partially hidden by the side navigation on desktop, the `.main-container` class needs to be added on the outside wrapper of your page's content.

## Navigation content

The default slot holds the navigation content. It must be a `<nav>` landmark with a descriptive `aria-label` and `<ul>`/`<li>` lists for the items. Nested levels are nested `<ul>`s. The section heading is optional.

**Icons should only be set on the first level elements.** For usability reasons, all or none of the level 1 elements should have an icon.

Expandable elements can either always be `post-collapsible` elements, or never be `post-collapsible` elements - in which case they're always expanded by default.

## Examples

The side navigation comes with two size variants:

- Large (default): for a flat and short navigation
- Small: for deep and long navigation (set adding the `is-small` property on the `<post-side-navigation>`)

## Accessibility

### Links and disclosure buttons

Use `<a>` for links and a `<button>` inside `<post-collapsible-trigger>` for collapsible sections. A link navigates, a button discloses; never nest one inside the other.

### Focus management

On mobile, focus moves into the navigation when the dialog opens and returns to the trigger when it closes. The rest of the page is inert while the dialog is open.
