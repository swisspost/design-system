### Automatic Positioning

By default, the `post-closebutton` positions itself automatically.
This means the component applies its own positioning styles to place the button at the top-right corner of the element it closes.

<post-banner type="info">
  For this to work correctly, the closable element must be the **offset parent** of the close button.
  The offset parent of an element is the nearest ancestor whose `position` is not `static`.
  If no such ancestor exists, the offset parent defaults to the `body`.

  To make the closable element the offset parent, give it a non-static position.
  You can do this by using utility classes such as `.position-relative` or `.position-absolute`.
</post-banner>

<Canvas sourceState="shown" of={CloseButtonStories.AutomaticPositioning} />

### Manual Positioning

If the available space around the closable element is insufficient for the default automatic positioning,
you can switch to manual positioning utsing the `position="manual"` property.

When `position="manual"` is used, no positioning styles are applied by the component.
The button is rendered inline like a regular button, and you are responsible for providing any styles needed to place it.

## Auto-close behavior

`<post-closebutton>` automatically walks up the DOM when clicked and closes the nearest supported ancestor.
No `onClick` handler or event listener is required on the consumer side.

### Supported closable targets

| Ancestor | Close method |
|---|---|
| `<post-banner>` | `dismiss()` |
| `<post-popover>` | `hide()` |
| `<post-popovercontainer>` | `hide()` |
| `<post-collapsible>` | `toggle(false)` |
| `<post-accordion-item>` | `toggle(false)` |
| `<dialog>` | `close()` |
| native `[popover]` element | `togglePopover(false)` |

### Three placement contexts

The auto-close behavior works regardless of where `<post-closebutton>` is placed relative to the closable element:

1. **Light DOM**  `<post-closebutton>` is placed directly inside the closable element (e.g. inside a `<dialog>`). Standard `parentElement` traversal reaches the ancestor directly.
2. **Slotted**  `<post-closebutton>` is projected into a component via a named slot (e.g. `<post-closebutton slot="close-button">` inside `<post-banner>`). The element stays in the light DOM, so the traversal still finds the banner host on the way up.
3. **Shadow DOM rendered**  `<post-closebutton>` lives inside a component's own shadow root (e.g. the internal close button of `<post-popover>`). When traversal reaches a shadow root boundary, it automatically jumps to the shadow host and continues upward.

In all three cases no consumer wiring is needed.
