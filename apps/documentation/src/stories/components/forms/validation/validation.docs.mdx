<div className="lead">
  The Validation component is a reusable component designed to provide consistent validation
  feedback across all components that require input validation.
</div>

Validation employs the `:invalid` and `:valid` pseudo-classes to provide styling for `<input>`, `<select>`, and `<textarea>` elements. To control when these styles are displayed, the `.was-validated` class must be applied to a parent element (e.g. `<form>`).

Alternatively, `.is-invalid` and `.is-valid` classes may be used instead of the pseudo-classes for server-side validation, without the `.was-validated` parent class.

These classes control the display of corresponding feedback messages that should be placed under the component: `<p className="valid-feedback">...</p>` or `<p className="invalid-feedback">...</p>`.

### Radio Button

Radio buttons are always used as part of a group. Validation is applied to the group as a whole, not to individual radio buttons.

This example demonstrates how a validation message can be programmatically associated with a single radio input. However, in real-world usage, validation should be implemented at the group level.

### Radio Button Group

Radio buttons are validated as a group (all inputs sharing the same `name`), not individually. Validation feedback must be linked to every radio input using `aria-describedby`.

When the group is `required` and no option is selected, `aria-invalid="true"` needs to be applied to all radio inputs.

The `required` attribute only needs to be set on one radio input in the group.
