## Installation

### Choose between lazy-loaded or bare component version

Internet-header package provide two versions:

- Lazy-loaded (default): it allows better performance by dynamically load the component when used, so it reduces bundle size, startup time and first paint.
- Bare component: if you already manage the lazy-loading or don't need it for any reason, you can use the component without any overhead.

## Migration from v9 to v10

This guide covers the breaking changes introduced in version 10 of the Internet Header.

### What changed

Version 10 reworks the component from the inside out.
`swisspost-internet-header` is now a thin wrapper that fetches the portal config and delegates all rendering to Design System components: `post-header`, `post-logo`, `post-language-menu`, `post-mainnavigation`, and others.

The public tag name (`swisspost-internet-header`) and the `project`, `environment`, `language`, `active-route`, and `full-width` props are unchanged.
Everything else — feature-toggle props, runtime overrides, and language detection helpers — has been removed.

### Support

If you need support during your migration,
don't hesitate to contact the [Design System Team](https://github.com/swisspost/design-system/discussions/categories/general) on GitHub.

### 1. Add the new required `text-*` props

Version 10 introduces a set of required visually hidden label props.
They are passed through to the inner Design System components and the component will throw if any are missing.
Provide translated values for every language your application supports.

See the full list of `text-*` props and their descriptions in the [Internet Header documentation](?path=/docs/ebb11274-091b-4cb7-9a3f-3e0451c9a865--docs).

### 2. Remove props that no longer exist

The following props have been removed and have no effect in v10.

<ul>
  <li><code>stickyness</code></li>

  <li><code>login</code></li>
  <li><code>search</code></li>
  <li><code>skiplinks</code></li>
  <li><code>config-proxy</code></li>
  <li><code>language-cookie-key</code></li>
  <li><code>language-local-storage-key</code></li>
  <li><code>logout-url</code></li>
  <li><code>self-admin-origin</code></li>
  <li><code>os-flyout-overrides</code></li>
  <li><code>custom-config</code></li>
  <li><code>language-switch-overrides</code></li>
</ul>



### 3. Update runtime usage

The `language`, `active-route`, `environment`, and `full-width` props are still reactive and can be set at runtime.
All other programmatic property assignments from v9 have no effect in v10 and should be removed.
