## Usage

Import the packages global styles. They are required for the components to work properly.

<Source code={GlobalCSSSample} language="scss" />

The package provides different entry points for CSR (Client-Side Rendering) and SSR (Server-Side Rendering). Each is available as a **monolith** import (all components in one bundle) or as **standalone** imports (individual components for better tree-shaking).

### Client side rendering (CSR)

If your project is purely client-side rendered, you can use the CSR entry points.

### Icons

The package includes generated React icon components, built from `@swisspost/design-system-icons`,
so it is no longer necessary to have the icons package installed as a dependency to use icons.
Under the hood, the `<post-icon>` component is being used to render the icons consistently,
therefore, all properties of this component are also available for the React icons.
See the [icon documentation](https://next.design-system.post.ch/?path=/docs/0dcfe3c0-bfc0-4107-b43b-7e9d825b805f--docs) for more information on it's features.

<post-banner variant="info">
The icon components are currently available only with built-in SSR compatibility. However, they can also be used in pure client-side projects.
</post-banner>
